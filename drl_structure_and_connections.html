Reading order: Config and state (config.py, state_action.py) underpin M1. Checkpoints feed platform_model_registry.py for M2. Portfolio transitions from M3 feed x_training.py (M4). cross_platform_drl_engine.py orchestrates M5 using x_model.py and per-platform paths through cross_platform_optimizer.py and hybrid_optimizer.py.

Adapters normalize each platform’s API; the optimization service layer exposes routes/engine that call into the shared DRL module (drl_integration.py and core agents).

6. Data, models, and demo outputs

  • Training_Data/ — input templates, synthetic CSVs, Criteo-style pretrain files, and outputs/ (single-platform actions, cross-platform JSON, platform test CSVs).
  • models/ — persisted runs (synthetic_run/, bq_run/) with epochs and training_info.json.
  • checkpoints/, m1_outputs/, m2_outputs/ — run logs and JSON summaries (m*_execution_log.txt at repo root).
  • demos/notebooks/ — Jupyter demos for single- vs cross-platform optimizers.
